# Session Notes: 2026-06-13

## Key Learnings

### ghstack resubmit doesn't exist
User explicitly corrected: `ghstack resubmit` is not a real command.
To update a stack: edit → `jj squash -r <rev>` → `ghstack` (re-submit entire stack).

### Commit message style
User prefers natural short sentences: "Fix some stuff about that", "Update stuffs".
Not conventional commits (`feat:`, `fix:`), not lowercase imperatives.
Capitalized, no prefix, no body, no trailers.

### ghstack land verification
After `ghstack land <URL>`, always verify changes actually landed on main.
Check with `git log main` or `jj log -r main`.

### ghstack only works on HEAD commit chain
`ghstack` submits the current HEAD commit chain. Detached HEADs or
non-contiguous commit ranges won't work. Always ensure commits are
linear and connected before submitting.

### Skills repo porting pattern
When local skills are updated, port changes to shikanime-studio/skills repo:
1. Copy updated SKILL.md and references/ to the repo
2. Update README.md catalog table
3. Commit with sentence-style message
4. Push to remote
5. Remove local duplicate to avoid name collisions with external_dirs

### Duplicate skill cleanup
After porting to skills repo, remove local copies to prevent
`skill_view` ambiguity errors. The skills repo version (loaded via
`external_dirs`) becomes the canonical version.
